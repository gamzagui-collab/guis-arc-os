# GUI's Arc OS Master Plan v10

## 1. 프로젝트 정의

GUI's Arc OS는 건설현장 운영체제(Construction Site Operating System)이다.
단순한 날씨 앱이나 정보 조회 사이트가 아니라, 현장소장·안전관리·품질관리·공사관리·자재/장비관리자가 오늘 해야 할 일을 빠르게 판단하고 실행할 수 있도록 돕는 업무 OS를 목표로 한다.

핵심 문장:

> 현장 관리자가 아침에 이 프로그램 하나만 켜면 오늘 해야 할 일을 5초 안에 파악할 수 있는 운영체제.

## 2. 핵심 원칙

1. 기존 기능 삭제 금지
2. Guest / 현장등록 / 현장접속 유지
3. 메뉴는 역할별로 분리
4. TODAY 입력값을 모든 메뉴가 공유
5. 정보는 요약 → 펼치기 구조
6. 공식 기준 / 현장 운영 기준 / AI 추천을 명확히 구분
7. 불확실한 정보는 Needs Verification으로 표시
8. UI는 가독성 우선, 모바일에서도 핵심 데이터가 먼저 보이도록 구성

## 3. 브랜드

- 이름: GUI's Arc OS
- 부제: Construction Site Operating System
- 긴 이름이나 Enterprise 명칭은 사용하지 않는다.

## 5. 데이터 흐름

현장 등록 또는 Guest 시작 → 오늘업무 입력 → 날씨/품질/안전/공사/자재장비/DB 자동 연결 → 역할별 체크리스트와 판단 생성.

## 6. 날씨 엔진 방향

- KMA / ECMWF / GFS / JMA 비교
- 6일 1시간 강수 비교표 유지
- 1시간 강수 그래프 유지
- 예보처 4곳은 동일한 하늘색 계열
- 평균/최대/일치도 열 구분
- 날짜 표기: 7/9 + (수)
- Leaflet 지도는 Windy 좌표 이동 목적
- 지도 클릭 시 Windy iframe도 동일 좌표로 이동
- AI 오늘 강수 의견은 그래프 위 전체 폭 카드로 표시

## 7. 품질관리 방향

품질관리의 핵심은 타설 등록이다.

타설 등록 → 공시체 자동 생성 → 시험일정 자동 생성 → 오늘 품질업무 생성 → 시험결과 입력 → 성적서 관리 → 보고서/PDF/QR 연동.

기본 공시체 템플릿:

- 수직부재 탈형강도: 3개
- 수평부재 탈형강도: 3개
- 필라서포트 확인: 3개
- 28일 압축강도 표준양생: 9개
- 28일 압축강도 현장양생: 9개
- 예비 공시체: 3개
- 120㎥당 기본 30개

주의: 최종 적용은 품질관리계획서, 특기시방서, 감리 지시, 최신 KCS/KS/품질관리 업무지침을 우선한다.

## 11. DB 방향

DB는 단순 검색이 아니라 각 메뉴의 판단 근거이다.

구분:

- Verified: 공식 확인
- Field Standard: 현장 운영 기본안
- AI Recommendation: AI 추천
- Needs Verification: 검증 필요
