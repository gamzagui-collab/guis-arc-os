# GUI's Arc OS v8.6 Master Architecture

## 1. 목표

GUI's Arc OS는 단순 날씨앱이 아니라 **건설현장 운영 보조 OS**입니다.

핵심 목표는 다음입니다.

- 현장 직원이 오늘 해야 할 일을 5초 안에 파악
- 일정 하나를 입력하면 안전·품질·공정·날씨·장비·자재·문서가 자동 연결
- 공식 기준 / 현장 운영 기본안 / AI 추천을 명확히 구분
- 긴 설명보다 요약, 체크리스트, 복사, 출력 중심
- DB는 계속 확장 가능하게 설계

---

## 2. 화면 철학

### 5초 요약
모든 화면의 최상단은 3~5줄 요약으로 구성합니다.

예:

```text
오늘 이것만 확인
1. 타설량 360㎥ → 공시체 기본 45개
2. 15시 강수 가능 → 보양재·배수로 확인
3. 펌프카 작업반경 통제 필요
```

### 펼치기 구조
상세 기준, 법령, KCS, KS, 사진 예시는 클릭해서 확인합니다.

### 신뢰도 표시
모든 지식 데이터는 아래 중 하나를 표시합니다.

- Verified: 공식 확인
- Field Standard: 현장 운영 기본안
- AI Recommendation: AI 추천
- Needs Verification: 검증 필요

## 3. 핵심 모듈

### Dashboard Engine
오늘 작업, 위험도, 날씨, 일정, AI 브리핑, 품질·안전 체크를 한눈에 표시합니다.

### Weather Engine
KMA / ECMWF / GFS / JMA 비교, 강수·풍속·온열·한랭 위험을 현장 작업 판단으로 변환합니다.

### CSI Engine
Construction Stress Index. 기온·습도·풍속·작업종류·작업위치·직사광선을 반영해 작업 체감위험도를 산출합니다.

### Quality Engine
공시체, 슬럼프, 공기량, 염화물, 크랙, 재료분리, 양생, 파일, 토공, 흙막이, 가시설 등 품질관리 업무를 규정과 연결합니다.

### Safety Engine
추락, 낙하, 끼임, 전도, 양중, 화재, 밀폐, 굴착, 폭염·한랭 등 안전관리 항목을 공종·장비·날씨와 연결합니다.

### Construction Engine
공종별 시공순서, 관리 포인트, 감리 지적 가능성, 사진기록 포인트를 제공합니다.

### Schedule Engine
현장일정 입력을 기반으로 오늘 필요한 정보만 자동 호출합니다.

### Equipment Engine
펌프카, 크레인, 지게차, 고소작업대, 백호우 등 장비별 작업계획과 점검사항을 관리합니다.

### Material Engine
자재반입, 검수, 보관, 하역, 품질서류, MSDS, 시험성적서 등을 관리합니다.

### AI Assistant Engine
일정·공종·날씨·DB를 종합해 역할별 할 일, TBM, 위험요인, 품질관리 요약을 생성합니다.

## 4. DB 설계 원칙

### Knowledge Node
모든 데이터는 노드로 관리합니다.

예:

```text
concrete_pouring
 ├─ specimen
 │   ├─ vertical_strip_strength
 │   ├─ horizontal_strip_strength
 │   └─ standard_28d_strength
 ├─ slump_test
 ├─ air_content_test
 ├─ curing
 ├─ rain_protection
 └─ pump_car
```

### Node 필드

```json
{
  "id": "concrete.specimen.standard_28d",
  "title": "28일 압축강도 공시체",
  "category": "quality",
  "summary": "시험빈도당 3조(9개)를 기본으로 산정",
  "confidence": "Verified",
  "sourceRefs": ["건설공사 품질관리 업무지침", "KS F 2403", "KS F 2405"],
  "whenToUse": ["콘크리트 타설"],
  "inputs": ["타설량", "배합", "타설일"],
  "outputs": ["공시체 개수", "시험일", "사진기록"],
  "why": "28일 압축강도 품질판정을 위한 기본 공시체",
  "checklist": []
}
```

## 6. 일정 기반 자동 호출 예시

### 입력
```text
7월 10일
콘크리트 타설
360㎥
펌프카 52M
슬래브
```

### 자동 생성
```text
품질
- 시험빈도 3회
- 공시체 기본 45개
- 슬럼프·공기량·염화물·온도 시험
- 현장양생 공시체 보관

안전
- 펌프카 아웃트리거
- 레미콘 차량 동선
- 타설구간 추락방지
- 강우 시 미끄럼

날씨
- 강수 위험 시간
- 풍속 위험
- 온열 CSI

공사
- 타설순서
- 이어치기 위치
- 다짐계획
- 양생계획

사진
- 검측 완료
- 송장
- 슬럼프
- 공시체 라벨
- 보양
```
