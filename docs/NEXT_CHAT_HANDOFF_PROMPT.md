# GUI's Arc OS 다음 채팅 인수인계

현재 Stable: **v9.8.8**

완료: 역할별 자동생성 규칙 위에 작업 시작조건·선행조건 판정 엔진을 추가했다. Work ID마다 안전·품질·공사·자재·장비의 필수조건을 교차 확인하고, 간편화면에는 시작 가능·확인 필요·시작 보류만 표시한다.

다음 권장: **v9.9 다중 사용자 협업 기반** — 원도급사 검토자 화면과 하도급사 사진·음성·수기도면 제출 화면을 분리하고, 제출자료를 기존 Work ID 자동생성 흐름에 연결한다.

핵심 원칙:
- 한 번 입력하면 다시 입력하지 않는다.
- 사용자는 행동만 하고 시스템은 증거를 만든다.
- 소규모 현장과 50대 이상 사용자를 기준으로 큰 버튼, 최소 입력, 자동 기록을 우선한다.
- Process DB → Work Instance DB → Task DB 통합 구조를 유지한다.
- 5단계 책임이력은 DB에 자동 저장하되 사용자 화면은 미조치·조치중·완료 3단계로 단순화한다.
